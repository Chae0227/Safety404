# 🧠 404 Safety Not Found  
### AI 안전불감의 시대: 사고(思考)가 사라진 순간을 체험하고, 다시 생각을 복원하는 플랫폼

AI는 생각을 대신해주는 도구가 되었고  
우리는 점점 **사고(思考)** 대신 **AI의 응답**을 믿기 시작했습니다.  

검색보다 AI에게 묻고,  
이해보다 요약을 요청하며,  
판단보다 “그럴듯한 답”에 만족하는 시대.

그 결과 우리는 **새로운 안전불감(Safety Blindness)** 속에 있습니다.  
사고(事故)가 아니라, **사고(思考)의 부재**가 위험이 되는 시대.

> “당신을 찾을 수 없습니다.”  
> 이 문구는 단순한 에러 메시지가 아니라  
> **생각을 잃어버린 나 자신**을 상징합니다.

404 Safety Not Found는  
AI 의존을 시각적으로 체험하고,  
AI의 정보 정확도를 스스로 검증함으로써  
다시 **‘생각하는 능력’을 회복**하도록 설계된 플랫폼입니다.

# 🎛 주요 기능 (Features)

## 1) 🔥 **AI 의존도 체험 (AI Interaction Simulation)**  
Gemini API와의 대화를 통해  
사용자의 **AI 의존도 게이지**가 실시간으로 상승합니다.  

게이지가 100%에 도달하면  
전체 UI가 글리치·파편화·붕괴되는  
**인터랙션 기반 ‘사고 붕괴 체험’**이 발생합니다.

이 연출은  
AI 의존이 극대화된 상태에서  
“사고(思考)가 사라짐”을 직관적으로 보여줍니다.

---

## 2) 🧐 **AI 답변 검증 시스템 (AI Fact Verification)**  
이 플랫폼은 **AI가 생성한 정보만 검증**합니다.  
(인터넷 임의 정보 검증 ❌)

흐름은 다음과 같습니다:

1. AI가 생성한 답변 중  
   “이건 맞나?” 싶은 내용을 선택  
2. 사용자가 직접 **근거 자료(책, 논문, 공식 문서 등)** 첨부  
3. 리뷰어가 근거·출처·논리로 검증  
4. 검증 통과 시  
   → **‘검증된 정보 게시판’**에 등록  
5. 사용자 + 리뷰어 모두 **포인트 지급**

즉,

### ✔ AI가 제공한 정보의 정확성  
### ✔ 인간의 사고/판단을 기반으로 재검증  
### ✔ 그 결과만 모아 신뢰 데이터베이스 생성  

이 3단계 구조가 Safety404의 핵심입니다.

---

## 3) 📚 **검증된 정보 데이터베이스 (Verified Knowledge DB)**  
사용자가 제출 → 리뷰어가 검증 → 통과한 정보만 축적됩니다.

구조는 다음과 같습니다:

**대분류 (카테고리)**  
- 의학  
- 과학  
- 수학  
- 컴퓨터/IT  
- 사회/경제  
- 생물/동물  
- 예술/일상  
- 엔터테인먼트  

**태그 기반 소분류 예시**  
- 백신, 감기, 통계, 확률, JAVA, Python, 알고리즘, 흉부 X-ray, 금리, 물가, 고양이 행동 등

심플하고 직관적인 태그 구조로  
누구나 쉽게 정보를 찾을 수 있는 UI를 제공합니다.

---

## 4) 🏅 **포인트 & 리뷰어 시스템**  
정보 생태계를 건강하게 유지하기 위해  
모든 참여자에게 포인트를 제공합니다.

### 사용자  
- AI 답변 검증 요청 → +1  
- 검증 통과 → +10  

### 리뷰어  
- 검증 1건 심사 → +3  
- 일정 포인트 이상 → 자동 리뷰어 승급  

포인트는 단순 보상을 넘어  
**신뢰 기반 정보 생태계를 만드는 동기부여 시스템**입니다.

404 Safety Not Found는

AI가 준 정보조차 직접 검증하게 함으로써  
우리 스스로 **정보를 판단하고 사고하는 감각을 회복**하게 만드는 플랫폼입니다.
